- **与 Knowledge Summary 冲突修复** → **【L4/seam】**

## **Rule Match**

- **核心定位、trigger / routing、EntityLog / RuleLog 分工、entity-level / pattern-level scope、non-LLM 执行语义、rule hit / miss / invalid handoff、JE Generator 边界、runtime-only / finalization 写入边界、rule-hit handoff** → **【L2】已收口**（见 `L2_proposals/Rule Match__L2提案.md`）
- **Rule 资格的内容判据**（全函数：scope 可客观判定 + 历史 treatment 唯一无分叉 + 输出 judgment-free 完整 + 事前已批准；次数 / 跨月 / 频率只是证据非门槛；repeated cases / 相似描述 / LLM reasoning / CaseLog 趋势 / 模糊语言只能进 candidate）→ **【L2】已收口**
- **同一 entity 下 rule 集合结构**（允许多条 active rule，互不重叠是写入时不变量；entity-level 与区分性 pattern-level 同 scope 互斥；"默认+例外"建模为互斥 pattern + catch-all；0/1/≥2 命中 = miss/hit/fail-closed blocked，不设 priority）→ **【L2】已收口**
- **RuleMatch 输入 handoff 契约面**（三桶语义：身份基础 entity_id / 放行基础 automation_policy 投影 / 客观交易事实；最终分类 = 命中 rule 的 approved treatment + 桶 C 交易事实，桶 C 必要不充分）→ **【L2】已收口**（exact 字段 schema → **【L3】**；assembler 谁组装 → **【L4/seam】**）
- **RuleLog 查询定位**：全系统以 entity_id 单坐标，RuleLog 按 entity_id 索引，RuleMatch 凭 entity_id 现查候选 active rule 集再用桶 C 比对；EntityLog 不存 rule_id 成员列表（成员关系单一归 RuleLog，避免双真值 / 漂移）→ **【L2】已收口（同时是对 RuleLog 设计的前置约束）**；entity_id 索引的存储 / 检索机制 → **【L4/seam】**
- **automation_policy → RuleMatch 放行**：由 EntityLog automation_policy 承载（不单设 supports_rule_match 字段），并经 ER handoff projection 传给 RuleMatch；"放行 RuleMatch"是其取值语义 → **【L2】已收口**；"哪种取值 = 允许 rule-based automation"的 exact 取值集合 → **【L3】**（RuleLog / EntityLog 联合，需对齐 Governance）
- **RuleMatchNode exact input / output fields、rule-hit handoff schema、rule_hit / rule_miss / blocked enum、accounting treatment（judgment-free 完整）schema、catch-all pattern 语法糖、资格阈值具体数字** → **【L3】**
- **RuleMatch matching algorithm、condition validation、amount range validation、rule 集合 overlap-validation、ER handoff / workflow router / RuleLog reader 调用顺序、多 log finalization、RuleLog 与 EntityLog 同步写入机制、rule hit count / rule health 统计维护机制** → **【L4/seam】**
- **Rule promotion eligibility、approval workflow、candidate queue、rule lifecycle、rule modification / downgrade / deletion、rule staleness / 前瞻有效性、case-derived rule promotion evidence governance contract** → **【L2·外阻】**（依赖 RuleLog / Governance）
- **下一步**：RuleMatch 节点侧 L2 已实质完成；BK_Copilot `entity_log` 草案已对齐"经 ER handoff 投影、不直接读 EntityLog"口径。闭环须先开 **RuleLog L1-L2**（资格判据的最终归属、entity_id 索引语义、automation_policy 取值语义、rule lifecycle 都在那里落地）。
