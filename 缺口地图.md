# 缺口地图

**标签**：**【L2】** 本轮收口 · **【L3】** 联合 L3 · **【L4/seam】** 记录层机制 · **【L2·外阻】** 圈外依赖

## **Entity Resolution**

- **stable 阈值** → **【L2】已收口**（见 `L2_proposals/Entity Resolution__L2提案.md`）
- **输出状态两态化 stable / unknown、unknown reason、stable reason、external evidence、exact Alias match、new_stable_entity 及时 publication** → **【L2】已收口**（见 `L2_proposals/Entity Resolution__L2提案.md`）
- **stable entity 确认后的 Entity Log publication 语义** → **【L2】已收口**（实际写入执行者 / 调用方式 / 顺序 → **【L4/seam】**）
- **entity_resolution_output 字段 schema** → **【L3】**
- **new_stable_entity provenance 字段 schema** → **【L3】**
- **knowledge_summary 冲突修复流程** → **【L2·外阻】**（依赖 Knowledge Summary / Governance）

## **Entity Log**

- **identity authority 主档案定位、Entity / Alias authority 与 projection 边界、五类保存集合、保存 / 不保存分流** → **【L2】已收口**（见 `L2_proposals/Entity Log__L2提案.md`）
- **stable entity 创建入口** → **【L2】已收口**（仅 ER `new_stable_entity` 与 accountant explicit identity confirmation；执行机制 → **【L4/seam】**）
- **stable entity lifecycle 最小状态** → **【L2】已收口**（active / merged / archived；exact enum / validation → **【L3】**）
- **automation_policy 的 entity 级控制语义** → **【L2】已收口**（当前控制状态归 Entity Log；存储 / Governance projection 形态与自动降级 exact contract → **【L4/seam】**）
- **merge / split 后 Entity Log 侧身份状态与重定向语义** → **【L2】已收口**（Alias / Case / Rule 跨 log 重判与执行机制 → **【L4/seam】**）
- **person 类型 stable entity 边界** → **【L2】已收口**（是否入 Entity Log 取决于是否经 ER / accountant confirmation 成为 stable entity；Profile ref / projection 形态 → **【L3 / L4】**）
- **entity_record / provenance 字段 schema** → **【L3】**
- **五类保存信息各自的 field schema、enum、refs 形态** → **【L3】**
- **alias_record / Alias 库技术形态、Entity Log 与 Alias Log 物理存储 / projection 边界** → **【L4】**
- **automation_policy 自动降级 mutation contract** → **【L4/seam】**

## **Alias Log**

- **stable entity 确认后 raw transaction surface text 的 Alias Log 写入资格** → **【L2】已收口**（见 `L2_proposals/Alias Log__L2提案.md`；执行机制 → **【L4/seam】**）
- **exact raw description match 直接支持 ER stable identity reuse** → **【L2】已收口**
- **non-exact / normalization / equivalence / vector / semantic matching 是否可直接支撑 stable identity** → **【L3 / L4】**（本轮 L2 已暂定只有 exact match 可直接支撑 stable）
- **多匹配边界** → **【L2】已收口**（Alias Log 不选 winner，交给 ER 判断 stable 或 unknown + reason）
- **merge / split 后 Alias 归属语义** → **【L2】已收口**（merge 跟随；split 治理重判；机制 **【L4/seam】**）
- **Alias 对下游的支持范围** → **【L2】已收口**（唯一直接 reader 是 ER，其他节点只能通过 ER 输出的 stable entity 间接受益）
- **alias_record 字段 schema** → **【L3】**
- **Alias 库技术形态、与 Entity Log 存储边界** → **【L4】**
- **写入责任 / 审批 / 统一 write 机制** → **【L4/seam】**
- **Alias mutation 与 Governance audit / correction / finalization 边界** → **【L4/seam】**

## **Case Log**

- **write eligibility** → **【L2】** ★
- **completed / corrected / reversed / split 的 supersession 语义** → **【L2】** ★（机制 **【L4】**）
- **record schema / field / enum / validation** → **【L3】**
- **exact writer / Transaction Log trigger order / 多 log 统一写入** → **【L4/seam】**
- **case-derived rule / automation / entity risk candidate governance contract** → **【L2·外阻】**
- **与 Knowledge Summary 冲突修复** → **【L4/seam】**
