# 缺口地图

**文件职责**：记录尚未收口、后续节点审计仍需要追踪的 L3 / L4 / seam / L2·外阻缺口。

**清理口径**：已确认且已经收口的 L1 / L2 不再写入本文；这些结论以 `BK_Copilot/` 下对应正式草案为准。

**标签**：**【L3】** 联合 L3 · **【L4】** 实现 / 存储机制 · **【L4/seam】** 跨层接缝机制 · **【L2·外阻】** 圈外依赖

## **Entity Resolution**

- **stable exact numeric threshold / confidence cutoff / 评分规则** → **【L3】**（如后续需要，不在 L2 冻结）
- **entity_resolution_output 字段 schema** → **【L3】**
- **new_stable_entity provenance 字段 schema** → **【L3】**
- **external evidence reference exact field schema（URL / source title / retrieved_at / snippet 等）** → **【L3】**
- **ER 判定新建 stable entity 后的实际写入执行者 / 调用方式 / 顺序** → **【L4/seam】**
- **external lookup retrieval / snapshot / cache 机制、source quality taxonomy** → **【L4/seam】**
- **knowledge_summary 冲突修复流程** → **【L2·外阻】**（依赖 Knowledge Summary / Governance）

## **Entity Log**

- **stable entity lifecycle exact enum / validation / 退出或回滚规则** → **【L3 / L4】**
- **entity_record / provenance 字段 schema** → **【L3】**
- **五类保存信息各自的 field schema、enum、refs 形态** → **【L3】**
- **Profile 与 Entity Log 的 ref / projection 形态** → **【L3 / L4】**
- **alias_record / Alias 库技术形态、Entity Log 与 Alias Log 物理存储 / projection 边界** → **【L4】**
- **stable entity 创建后的实际写入执行者 / 调用方式 / 写入顺序 / 多 log finalization 机制** → **【L4/seam】**
- **automation_policy 存储 / Governance projection 形态** → **【L4/seam】**
- **automation_policy 自动降级 mutation contract** → **【L4/seam】**
- **merge / split 后 Alias / Case / Rule 跨 log 重判、迁移、阻断、finalization、audit trace 和回滚机制** → **【L4/seam】**

## **Alias Log**

- **non-exact / normalization / equivalence / vector / semantic matching 是否可直接支撑 stable identity** → **【L3 / L4】**
- **alias_record 字段 schema** → **【L3】**
- **Alias 库技术形态、与 Entity Log 存储边界** → **【L4】**
- **写入执行者 / 确切时机 / 顺序 / 统一 write 或 finalization 机制** → **【L4/seam】**
- **Entity merge / split 时 Alias Log 同步更新、split 后逐条重新归属、batch 操作、correction 审批和 audit trace** → **【L4/seam】**
- **Alias mutation 与 Governance audit / correction / finalization 边界** → **【L4/seam】**

## **Case Log**

- **record schema / field / enum / validation** → **【L3】**
- **`use_level` / `confirmed_by` exact enum** → **【L3】**
- **共享 accounting outcome（COA / HST / split / allocation）的全局字段结构** → **【L3】**
- **exact writer / Transaction Log trigger order / 多 log 统一写入** → **【L4/seam】**
- **Case Judgment 读取时的 per-entity 固定化聚合（rollup / 轻量展示层 / 检索工具或 subagent）** → **【L4/seam】**（读取层聚合机制；聚合内容全部由逐笔 case 派生，不作为 Case Log 存储字段）
- **supersession 的血缘链接字段与 corrected / reversed / split 后的重判执行机制** → **【L4/seam】**
- **merge / split / archive 后旧 case 重新归属的治理执行机制** → **【L4/seam】**
- **case-derived rule / automation / entity risk candidate governance contract** → **【L2·外阻】**
- **entity_level Knowledge Summary 与 Case Judgment 的 co-read source memory / 注入 contract** → **【L2·外阻/seam】**
- **与 Knowledge Summary 冲突修复** → **【L4/seam】**

## **Rule Match**

- **RuleMatch 输入 handoff exact 字段 schema** → **【L3】**
- **RuleMatch handoff 由谁组装 / assembler seam** → **【L4/seam】**
- **RuleMatchNode exact input / output fields、rule-hit handoff schema、rule_hit / rule_miss / blocked enum、accounting treatment（judgment-free 完整）schema、catch-all pattern 语法糖、资格阈值具体数字** → **【L3】**
- **automation_policy 放行 RuleMatch 的 exact enum** → **【L3】**（EntityLog / RuleLog / Governance 联合）
- **JE Generator 接口和 approved accounting treatment 可执行完整性口径** → **【L3 / JE Generator】**
- **RuleMatch matching algorithm、condition validation、amount range validation、rule 集合 overlap-validation、ER handoff / workflow router / RuleLog reader 调用顺序、多 log finalization、RuleLog 与 EntityLog 同步写入机制、rule hit count / rule health 统计维护机制** → **【L4/seam】**
- **RuleLog 按 `entity_id` 索引的存储 / 检索机制** → **【L4/seam】**
- **approval workflow、candidate queue 机制、modification / downgrade / deletion、rule staleness / 前瞻有效性、case-derived rule promotion evidence governance contract** → **【L2·外阻 / L4/seam】**

## **Rule Log**

- **rule schema、rule ref / rule_id 形态、scope enum、execution-state 字段名 / enum、source / approval provenance refs 形态** → **【L3】**
- **condition schema / enum、catch-all 语法糖、overlap-validation、condition satisfiability、amount range validation、冲突 repair path** → **【L3 / L4 / Governance】**
- **approved treatment exact shape、COA / HST / GST / split / allocation schema、JE Generator contract** → **【L3 / JE Generator】**
- **automation_policy 放行 RuleMatch 的 exact enum** → **【L3】**（EntityLog / RuleLog / Governance 联合）
- **RuleLog execution query contract、reader / assembler 调用顺序、projection / cache / permission 边界、`entity_id` 索引存储 / 检索 / 维护机制** → **【L4/seam】**
- **rule candidate queue、system-proposed promotion approval workflow、CaseLogEvidence 打包格式、accountant direct write 机制** → **【L2·外阻 / L4 / Governance】**
- **rule modification / overwrite / versioning / supersession / deletion / restore mutation path、跨 log lifecycle 命名统一、merge / split 后 rule 重判 / 迁移 / 回滚 / 治理 trace** → **【L2·外阻 / L3 / L4 / Governance】**
- **rule-hit ref 在 Transaction Log / Case Log 的字段形态、统计派生 / rollup / cache / maintenance job** → **【L3 / L4/seam】**
