# 系统上下文地图

## 文件职责

`系统上下文地图.md` 是新窗口理解系统材料的速读地图。

一句话职责：告诉新窗口“材料大概怎么分层、哪些来源有效、哪些目录需要用户授权才可读取”。

它只负责材料导航，不定义 node、field、memory layer 或 workflow 的正式边界，不替代 `BK_Copilot/`、`缺口地图.md`、主题记录或用户指定的长期结论文档。

## 当前使用规则

如需判断操作纪律、authority 或停止条件，读 `AGENTS.md`。

如需判断当前任务、下一步或 active risks，读 `当前任务状态.md`。

如需判断某个 node / memory layer 的当前正式边界，读 `BK_Copilot/` 下对应正式草案。

如需查看尚未收口的问题，读 `缺口地图.md`；该文件只索引未冻结 L3 / L4 / seam / L2·外阻缺口。

以下目录不进入 source authority 顺序：

- `new system/`
- `old_system_nodedesign/`

`new system/` 已作废，不再作为审计目标。`new system/` 与 `old_system_nodedesign/` 均为授权读取材料：未经用户明确同意，不得读取其文档内容。即使用户授权读取，也只能用于历史追溯或用户指定比较，不能用作权威参考、正确性证明、baseline、历史意图权威或可复用约束来源。

## New / Old 材料边界

本地图不再提供 New System / Old-System 的内容导航。New System 已作废，不再作为审计目标；Old-System 也不作为审计目标、baseline 或历史意图权威。

若任务确需读取 New System 或 Old-System 文档，必须先取得用户对具体读取范围的明确同意。未获同意时，新窗口只读取现行 BKCopilot 内容：active 治理 / 跟踪文档、`BK_Copilot/` 正式草案、模板规则、必要的 `L2_proposals/` 与用户指定主题记录。

## 使用边界

正确用法：

1. 用本文件快速知道系统材料分布。
2. 回到 `当前任务状态.md` 确认当前任务。
3. 按任务读取 `BK_Copilot/`、`L2_proposals/`、主题记录；只有在用户明确授权具体范围后，才可读取 `new system/` / `old_system_nodedesign/`。
4. 把未冻结缺口写入 `缺口地图.md` 或对应正式草案的 Open Boundary。
5. 把长期性结论写入用户指定的长期结论文档或对应正式草案，不写进本摘要。
