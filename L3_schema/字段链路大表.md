# 字段链路大表（跨节点字段真值唯一权威）

> **作用**：横着读——每个值从生到死一个名字、有出生有落点、含动作。这是跨节点字段真值的**唯一权威**；各节点 `{节点}_L3_Schema.md` 的字段指向本表，正式 `03 / M3` 由它派生。
> **状态**：draft，沿主链逐段建。本轮诞生第一段 **Evidence Intake → Entity Resolution**。
> **动作词**：`create`（凭空新建）/ `remake`（从上游算出新名新值）/ `pass`（原样透传）/ `read`（读用不改）/ `set`（同名敲定）/ `save`（落库）/ `—`（未出现）。
> **列**：值 ｜ Evidence Intake ｜ Entity Resolution ｜ 出生 → 落点。后续节点（Rule Match / Case Judgment …）通了再加列。

## 二、口径与断头检查

- **同值同名**：`raw_description` 在 EI / ER 同名透传，不漂移；运行期表面文字只此一名，`alias` 仅为其写入两个 log 时的存储记录名（不是运行期改名）。
- **`identity_evidence_refs` 为何不直接用 `evidence_refs`**：它是身份子集 + 并入 C 的外部来源 ref（EI 的 `evidence_refs` 无外部来源），属 `remake`，故单列。
- **每个值有出生有落点**：上表无断头线、无无主字段。
- **`direction` 枚举口径**：EI 标注「内部取值口径待与 Rule Match / JE 统一」，跨节点统一留后续段处理。
